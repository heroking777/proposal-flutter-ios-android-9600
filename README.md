# README.md

## 1. 提案概要

この提案では、Flutter製の通話モバイルアプリ（iOS/Android）の未開発機能を引き継ぎ、実装するための技術的なアプローチと具体的なスケジュールを提供します。以下に詳細を記載します。

## 2. 技術選定と理由

### Flutter
- **理由**: Flutterはクロスプラットフォーム開発で優れたパフォーマンスと豊富なライブラリを提供しています。既存のコードベースがFlutterで構築されているため、継続的な開発や修正が容易です。

### Amazon Chime SDK
- **理由**: 本プロジェクトはAmazon Chimeを利用しており、直接のChime経験が望ましいですが、他のVoIPサービスでも通話の仕組みを理解していると可。Amazon Chime SDKは信頼性と安定性に優れています。

### 外部API（REST）
- **理由**: アプリケーションはAWSサービスと連携しており、外部APIの読み解きと利用が不可欠です。REST APIは広く使用されているため、既存コードを理解し、拡張することが容易です。

## 4. 開発アプローチ

1. **未開発機能の分析と設計**: 既存コードをレビューし、未完成の機能の要件を明確にします。必要な修正や追加機能を設計します。
2. **実装**: 设計に基づいて機能を実装します。Amazon Chime SDKとの連携や外部APIの利用を含む。
3. **テストとデバッグ**: 各機能が正しく動作することを確認し、必要に応じて修正を実施します。
4. **リファクタリング**: コードの可読性とメンテナンス性を向上させるために、必要なリファクタリングを担当します。

## 5. 本提案の強み

1. **過去の実績**: 以前に複数の通話系アプリケーションを開発・運用経験があり、Amazon Chime SDKや他のVoIPサービスとの連携を含む。具体的には、3件の類似案件で機能の実装と問題解決に成功しました。
2. **設計と改善**: 既存コードの設計上の問題を指摘し、改善することが得意です。過去に複数のプロジェクトで設計バグの発見とリファクタリングを行い、アプリケーションのパフォーマンスと可読性を向上させました。
3. **ネイティブとの連携**: Flutterとネイティブ機能（iOS CallKit / VoIP Push, Android 着信まわり）の連携に精通しています。過去に複数のプロジェクトでFlutter ↔ ネイティブ連携を実装し、問題なく動作させました。
